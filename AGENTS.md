# AGENTS

Репозиторий ведется как документационный проект по мобильной фотографии на Redmi Note 12 Pro+.

## Skills

- `claude-plan-advisor` используется как **глобальный skill Codex**.
- Локальную папку `skills/` в этот репозиторий не добавлять.
- Skill должен применяться автоматически для:
  - проверки планов перед выполнением;
  - проверки хода выполнения;
  - проверки результата перед завершением сессии.

## Цель

- Держать `README.md` главной страницей мануала
- Поддерживать практичные, короткие инструкции без воды
- Всегда сохранять состояние работы для быстрого продолжения

## Источник истины

- Главная страница: `README.md`
- Статус работ: `workflow/CURRENT_STATE.md`
- Очередь задач: `workflow/BACKLOG.md`
- Передача контекста: `workflow/HANDOFF.md`

## Правила для изменений

- Основной пользовательский контент вести в `README.md` (single-page формат)
- Весь пользовательский контент ведется в `README.md` (папка `guide/` удалена)
- Примеры/исходники складывать в `assets/photos` или `assets/examples`
- После каждой заметной сессии обновлять `CURRENT_STATE.md`, `SESSION_LOG.md`, `HANDOFF.md`

## Формат работы (обязателен)

1. Обновить backlog при изменении приоритетов
2. Проверить план/прогресс/результат через глобальный `claude-plan-advisor` (авто)
3. Выполнить 1-2 логических шага по мануалу
4. Отразить, что сделано и что дальше, в `workflow/`
5. Оставить репозиторий в состоянии, удобном для передачи другому агенту

## Коммиты

  - `docs: add portrait shooting workflow`
  - `docs: refine night mode checklist`
  - `checkpoint: update current state and handoff`

## Критерий готовности секции

- Есть понятная цель секции
- Есть пошаговые настройки/действия
- Есть типичные ошибки и как исправить
- Секция встроена в `README.md` или имеет явную ссылку из него
