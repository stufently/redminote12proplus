# DECISIONS

## D-001 (2026-03-01)

- Тема: Главная страница мануала
- Решение: Центральной точкой входа остается `README.md`, а все длинные разделы выносятся в `guide/`
- Причина: Быстрый вход и единая навигация для любого агента

## D-002 (2026-03-01)

- Тема: Передача контекста между агентами
- Решение: Использовать фиксированный набор `workflow/BACKLOG.md`, `workflow/CURRENT_STATE.md`, `workflow/HANDOFF.md`, `workflow/SESSION_LOG.md`
- Причина: Можно безопасно прерывать работу и продолжать без потери контекста

## D-003 (2026-03-01)

- Тема: Формат изменений
- Решение: Держать изменения маленькими и завершаемыми в рамках сессии, закрывая их `checkpoint`-коммитом
- Причина: Упрощает передачу состояния и откат при необходимости

## D-004 (2026-03-01)

- Тема: Внешний second opinion по плану
- Решение: Использовать глобальный skill `claude-plan-advisor` в Codex без хранения локальной папки `skills/` в репозитории
- Причина: Быстрое улучшение планов и приоритезации без ручной сборки контекста

## D-005 (2026-03-01)

- Тема: Process-правила в `README.md`
- Решение: Убрать process-heavy блоки из `README.md` и вынести их в `workflow/PROCESS.md`
- Причина: Главная страница мануала должна быть удобной для чтения, а не перегруженной внутренним процессом

## D-006 (2026-03-01)

- Тема: Навигация по мануалу
- Решение: Перестроить вход через задачи пользователя и добавить `guide/INDEX.md` с маршрутами
- Причина: Линейное оглавление оказалось неудобным для быстрого входа

## D-007 (2026-03-01)

- Тема: Визуальные подсказки в разделах
- Решение: Добавить встроенные SVG-схемы "куда нажимать" и превью реальных кадров
- Причина: Без картинок сложно быстро понять шаги интерфейса и проверки результата
